JedEye

The JedEye is an Arduino-based handheld instrument designed for dry cave surveying. It measures azimuth (compass), clino, roll, and direct distance (lidar). A full user manual is published at https://sebkister.github.io/JedEye-Documentation/.

Internally, the JedEye runs on a Nano RP2040 Connect with a u-blox NINA-W102 radio module, a Meskernel LDJ-200 lidar, an LSM6DSOX IMU, an RM3100 magnetometer, and a 128x128 color OLED. It is operated through two buttons labelled Next (move the menu cursor) and Select (activate). USB-C is used for charging and firmware updates.

Bluetooth pairing
The JedEye uses Bluetooth Low Energy only - there is no PIN-based pairing. To make it visible to TopoDroid, switch the device to its Distance Meter screen via the on-device menu. It then advertises as JedEye_<serial>, where <serial> is the 16-hex-digit unique ID of the underlying Nano RP2040 Connect (e.g. JedEye_009315501C7B8A58).

In TopoDroid, open the Devices activity, tap Add new device, and start a BLE scan. The JedEye appears in the list with model "JedEye". Tap it to add it; TopoDroid records the MAC address and the device type. The pairing only needs to be done once - subsequent connections are automatic when the device is back in the Distance Meter screen.

Taking a shot
On the Distance Meter screen the OLED shows the current lidar distance and the current heading and clino in real time. A small Bluetooth icon in the bottom-right corner indicates that BLE advertising is active.

To capture a measurement, point the device at the target and invoke Measure with the Next / Select buttons. The captured values (distance, heading, clino, roll) are frozen on the OLED as the "saved" reading and simultaneously streamed to TopoDroid as a single shot. If TopoDroid's shot window is open, the new shot appears immediately in the list and is auto-assigned a station name like any other leg shot.

TopoDroid can also send a remote MEASURE command (over the BLE command characteristic) to trigger a shot from the app side - useful when the device is mounted on a tripod or otherwise out of reach.

Connection mode
The JedEye is always connected in continuous mode. Each measurement is transferred to TopoDroid as soon as it is taken. Shots taken while the device is not connected are not buffered or re-transmitted later - the JedEye does not currently expose its internal survey memory over BLE. The device is therefore best used in sessions where it can remain connected to the Android throughout.

Battery
The JedEye runs on an internal battery and charges over USB-C. The current battery level is shown in the device's Info screen and as a small icon on the main display.

Firmware notes
The JedEye firmware speaks a BLE protocol that reuses TopoDroid's SAP6 shot-frame format (a 17-byte notification: sequence byte plus four little-endian float32 values - bearing, clino, roll, and distance in metres), but exposes it under JedEye-specific 128-bit GATT service UUIDs. The on-device firmware does not implement the SAP6 ACK handshake; TopoDroid takes this into account and does not write per-shot ACKs to JedEye devices.
