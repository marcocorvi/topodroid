[6] CALIBRATION WINDOW

Tapping the Calibration menu in the Device activity window you go to the Calibrations dialog, which lists the calibrations of the selected DistoX. To create a new calibration tap on the New Calibration button in the dialog. This brings you to the window for the calibration informations. The "date" field is prefilled with the current date. The "device" field is prefilled with the address of the DistoX, and cannot be modified. You must give a "name" to the calibration. The name is what appears in the list of calibrations. You can optionally enter a brief "description". By default TopoDroid will pick automatically the calibration algorithm based on the DistoX model and firmware. However you can force it to use either the linear or the non-linear algorithm. When you are done tap the Save button (the leftmost) to save the calibration info to the database.

WARNING. The name of the calibration must not contain the character '/' (slash), which is the pathname separator.

  • Save saves the calibration informations to the database

    • CALIBRATION SETTINGS

  • Group policy: how to split the calibration data into groups. The three policies are described in the section about calibration data
  • Group tolerance: the maximum angular distance between consecutive data to be in the same group when the "distance-based" group-policy is used
  • Error: the error below which the computation of the coefficients teminates
  • Iterations: the maximum number of iteration for the calibration algorithm
  • Raw data: whether to display also the raw data. By default TopoDroid displays the azimuth, inclination, and roll. Check this option if you want to see also the raw calibration data
  • Algorithm: can be 0 (auto), 1 (linear), 2 (non-linear). If a calibration algo is left "auto", and this setting is not "auto", then the setting algorithm is used to calculate the calibration. If they are both "auto" the "linear" algorithm is used

    • When you open an existing calibration (tapping the calibration name in the list of the calibration of a DistoX), you get the same interface. The only difference is that the name of the calibration is no more editable.

    HOW TO CALIBRATE A DISTOX

    The DistoX needs to be calibrated before using. The calibration is a transformation of the data that accounts for the non-exact orthogonality of the axis of the sensors and effects of the electronics on the magnetic field. To compute the coefficients of this transformation, you must take a number of shots covering all the directions, and recording the raw readings of the sensors. This occurs when the DistoX is in "calibration mode".
    At a minimum you must take four shots (with different roll) in four directions at 90 degrees in the horizontal plane, as if these directions were going from the center of a cube towards the center of four side-faces, and eight shots from the center of the cube to the eight vertices. With TopoDroid it is possible to take four shots (at different rolls) also for the other directions besides the first four, and even take more than four shots.
    Before computing the calibration you must tell the program the groups of shots for each direction, ie, you must assign a group to each shot. The groups are denoted with positive integers. A shot with group "0" (zero) or no group, is not used to compute the calibration. Deleted shot have negative group number.
    TopoDroid implements three policies to assign groups automatically:
  • "four groups of four shots, followed by single shots": this is the default
  • "each group with four shots"
  • "shot group assignment based on the angular discrepancy between shots"
    •
    To summarize, to calibrate the DistoX you need:
  • turn the DistoX in calibration mode
  • take the shots of the calibration data
  • download the shots to the app
  • assign the groups to the shots (automatically or manually)
  • compute the calibration coefficients
  • upload the coefficients to the DistoX
  • revert the DistoX to normal mode

    Now fill the necessary fields, save the calibration info to the database, and open it. You are in the calibration data window, ready to take the calibration shots.
